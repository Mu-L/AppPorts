# 外部ストレージガイド

## 推奨構成

| 構成 | 推奨値 | 説明 |
|------|--------|------|
| 容量 | 256 GB 以上 | 移行するアプリの数に依存 |
| インターフェース | USB 3.0 以上 / Thunderbolt | USB 2.0 は遅い；大容量アプリの移行に時間がかかる |
| ファイルシステム | APFS | クローン、スナップショットをサポートし、最高のパフォーマンス |

## インターフェース性能比較

| インターフェース | 理論速度 | 実際の移行速度 | ユースケース |
|----------------|---------|--------------|------------|
| USB 2.0 | 480 Mbps | ~30 MB/s | 非推奨；遅すぎる |
| USB 3.0（USB-A） | 5 Gbps | ~350 MB/s | ほぼ十分 |
| USB 3.1 Gen 2（USB-C） | 10 Gbps | ~700 MB/s | 推奨 |
| Thunderbolt 3/4 | 40 Gbps | ~2500 MB/s | 最高のパフォーマンス |
| NVMe（Thunderbolt） | 40 Gbps | ~2800 MB/s | 最高のパフォーマンス |

## ファイルシステムの推奨

### APFS（推奨）

- クローン、スナップショット、スペース共有をサポート
- 最高のパフォーマンス、特に SSD において
- macOS ネイティブサポート

### HFS+

- 互換性が高く、古い Mac に適している
- クローンとスナップショットをサポートしない
- メカニカルハードドライブに適している

### exFAT

- クロスプラットフォーム対応（macOS + Windows）
- ハードリンクとクローンをサポートしない
- パフォーマンスが比較的低い
- 複数のシステム間で使用するシナリオに適している

## 容量計画

移行後の AppPorts の外部ストレージ使用量は、移行されたアプリとデータディレクトリのサイズに依存します。以下は一般的なアプリの参考サイズです：

| アプリタイプ | サイズ |
|-------------|--------|
| Chrome | ~500 MB |
| Microsoft Office | ~5 GB |
| Adobe Creative Cloud | ~20-50 GB |
| Xcode | ~15 GB |
| Final Cut Pro | ~5 GB |
| ローカル大規模言語モデル（Ollama） | ~4-30 GB |

::: tip 💡 容量の推奨
- 軽い使用（5〜10 アプリ）: 128 GB
- 中程度の使用（10〜20 アプリ）: 256 GB
- 重い使用（20 アプリ以上 + データディレクトリ）: 512 GB 以上
:::

## 備考

- 外部ストレージは接続したままにする必要があります；移行済みのアプリとデータディレクトリはオフラインでは使用できません
- 外部ストレージ上のデータは定期的にバックアップしてください
- 移行中に外部ストレージを抜かないでください
- 外部ストレージが故障した場合、AppPorts でアプリをローカルに戻すことができます
