# 外部存储指南

## 推荐配置

| 配置项 | 推荐值 | 说明 |
|--------|--------|------|
| 容量 | 256 GB 及以上 | 视迁移应用数量而定 |
| 接口 | USB 3.0 及以上 / Thunderbolt | USB 2.0 速度较慢，大型应用迁移耗时较长 |
| 文件系统 | APFS | 支持克隆、快照，性能最佳 |

## 接口性能对比

| 接口 | 理论速度 | 实际迁移速度 | 适用场景 |
|------|----------|-------------|----------|
| USB 2.0 | 480 Mbps | ~30 MB/s | 不推荐，速度过慢 |
| USB 3.0 (USB-A) | 5 Gbps | ~350 MB/s | 基本够用 |
| USB 3.1 Gen 2 (USB-C) | 10 Gbps | ~700 MB/s | 推荐 |
| Thunderbolt 3/4 | 40 Gbps | ~2500 MB/s | 最佳性能 |
| NVMe (Thunderbolt) | 40 Gbps | ~2800 MB/s | 最佳性能 |

## 文件系统建议

### APFS（推荐）

- 支持克隆、快照、空间共享
- 性能最佳，尤其适合 SSD
- macOS 原生支持

### HFS+

- 兼容性好，适合旧款 Mac
- 不支持克隆和快照
- 适合机械硬盘

### exFAT

- 跨平台兼容（macOS + Windows）
- 不支持硬链接和克隆
- 性能相对较低
- 适合需要在多系统间使用的场景

## 容量规划

AppPorts 迁移后的外部存储使用量取决于迁移的应用和数据目录大小。以下为常见应用的参考大小：

| 应用类型 | 大小 |
|----------|------|
| Chrome | ~500 MB |
| Microsoft Office | ~5 GB |
| Adobe Creative Cloud | ~20-50 GB |
| Xcode | ~15 GB |
| Final Cut Pro | ~5 GB |
| 本地大语言模型（Ollama） | ~4-30 GB |

::: tip 💡 容量建议
- 轻度使用（5-10 个应用）：128 GB
- 中度使用（10-20 个应用）：256 GB
- 重度使用（20+ 应用 + 数据目录）：512 GB 及以上
:::

## 注意事项

- 外部存储需保持连接，已迁移的应用和数据目录在离线时无法使用
- 建议定期备份外部存储上的数据
- 避免在迁移过程中拔出外部存储
- 如果外部存储出现故障，可通过 AppPorts 将应用迁回本地
